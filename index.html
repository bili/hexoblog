Vue2.x源码学习（1）视图和数据对象是如何关联的？

发表于 2019-02-15

vue最为人所知的特色就是数据变化会自动触发视图的更新，这里面最核心的原理就是利用了Object.defineProperty()方法，实现对数据变化的监听，这也是为什么vue不支持那些老旧浏览器的原因之一。

我们先看一下数据部分是如何在代码中流通的，知道了它的行走路径也就明白了它和视图的关系。本文我只关注data的部分。

data是如何实现响应的？

首先要介绍的是initState()，initState()是处理有关state的方法，包括props、methods、data、computed和watch。而data的相关逻辑则是在initData()中实现。

在initData()中，会先遍历data中的属性，将data中的属性全部代理到vue实例上，实现通过vue实例直接访问data的属性的效果，例如data: {a: 1, b: 2}，就可以实现直接通过vm.a访问，而不必vm._data.a了。完成这步后，则会为data创建一个Observer，它的作用就是为data中的所有属性添加响应功能，核心方法就是Object.defineProperty()，使得属性变化可以被监测到，并触发视图更新，这也是最关键的一步，从此，数据和视图的命运就被紧密地关联在一起了。

什么都可以学习

发表于 2018-09-29

比如我经常下班回来还要做饭，虽然会花去不少时间，有时也会抱怨偷懒，但是仍然享受烹饪带来的乐趣，那种将不同的食材组合在一起却又产生出超出原材料的诱人的味道的过程实在很吸引我。又比如我为了减肥，放弃了午休，大热天的在外面暴走，收获的不止是卡路里的消耗，更是向周围学习的机会。大量的城间徒步能让我认识更多的路，邂逅更多的风景。还比如我独自接了一个外包，项目虽然复杂困难，但却是一次难得的掌握新技术的实践机会。所以祸兮福之所倚，秉着福祸相依的出世态度才能学习成长。

前面举的几个例子也并不是暗示“要得必有失”的道理，事实上，还有不少是属于空手套白狼（原谅我词库干涸，找不到合适的词汇）的事例。每个人身边都有些家人朋友同事，甚至是陌生人。有些人你会发现善于解决问题，那你可以学习他解决问题的思路。有些人看东西很透，你可以向他请教。有些人穿着打扮得体舒适，你可以参考她的风格。有些人平时吊儿郎当，但是运动起来简直帅呆，你也可以去买个滑板拉风。人无完人，每个人身上多多少少都有我们值得参考效仿的地方。

还有些情况，遇到不懂的事情，你会去请教别人，请教网络，如果把这些总结记录下来，时不时回头看看，以后的你会感谢现在的你。

太多的人，太多的东西值得我们去学习。如果你羡慕什么，那就去学吧。一切皆可学习。也许你做不到学习快乐，但一定会因为学成而快乐。


Send2Phone小记

发表于 2018-09-29

Send2Phone，“人”如其名，就是把电脑上正在访问的网页地址一键推送到手机上。说实话这个功能早已经烂大街了，不过作为一个合格的程序员，造轮子是最爱干的事。

技术原理也很简单，就是利用了firefox上的油猴插件和手机端的 IFTTT 的 webhook 功能。

说到这，估计大家一下就明白了。

效果是这样的：每个访问的页面的左下角都躲着一个按钮，点击一下，就推送到手机了，实现自己的 hand off 功能。

配置IFTTT

访问 https://ifttt.com/maker_webhooks，进到触发器的创建页面。

这里面只要设置一下【事件名】就可以了，因为我要传的是动态的变量，所以这里的参数可以不用写。然后你可以直接把这个请求地址粘贴到浏览器访问一下，如果手机有收到推送，说明ok。

我的 git-flow 实操经验

发表于 2018-07-30


Git 对于项目开发/版本管理的好处就不用我多说了，别人也说的很多了，你可以随便搜索看一下。git 虽然好，但是如果利用不好，反而会让自己手忙脚乱，降低了开发效率。而 git-flow 就是一个成熟的开发流程（git-flow 其实是 git 命令的集合），我们只要跟着这套规范走，就可以少走不少弯路。下面就讲一下 git-flow 在我公司实际的应用和遇到的问题以及解决的方法。

先简单介绍一下 git-flow 中的各种分支：

分支

master

对应线上版本，唯一的主分支，最稳定。后面开发的代码通常是不能直接提交到主分支的，只有hotfix和release才可以。另外，推的时候最好打标签，方便以后回归和回顾。

develop

这个分支就是我们平常开发的代码库，大部分代码都会基于这个分支。

feature

如果有新功能，就需要开这个临时分支。这个是从 develop 分支拉出来的，基本上，一个功能就开一个 feature 分支，功能完成并提交合并到 develop 分支后，就可以把它删除了。
